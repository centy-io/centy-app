# Replace docs-specific pages with a generic slug-based dynamic route

Currently the app has hardcoded routes for docs:

- `/[org]/[project]/docs` → `DocsList`
- `/[org]/[project]/docs/[slug]` → `DocDetail`

We want to generalize this so any item type configured with `identifier: slug` is handled by a single dynamic route, the same way docs work today. The docs-specific pages should be removed in favor of this generic handler.

## Goal

Replace `/[org]/[project]/docs/*` with a generic `/[org]/[project]/[itemType]/*` (or equivalent) that:

- Works for any collection whose config has `identifier: slug`
- Renders a list page and a detail page (by slug), just like the current docs pages do
- Removes the docs-specific `DocsList` / `DocDetail` pages and replaces them with item-type-agnostic equivalents

## Acceptance Criteria

- [ ] A dynamic route handles list and detail views for any slug-based item type
- [ ] The `/docs` and `/docs/[slug]` hardcoded pages are removed (or redirected)
- [ ] Visiting `/[org]/[project]/docs` still works via the new generic route
- [ ] Visiting `/[org]/[project]/stories` (or any other slug-based type) works the same way
- [ ] Item detail pages navigate by slug — no numeric IDs or UUIDs in URLs
- [ ] Unit tests cover the generic list and detail rendering
- [ ] E2E tests cover navigation for at least one slug-based item type
