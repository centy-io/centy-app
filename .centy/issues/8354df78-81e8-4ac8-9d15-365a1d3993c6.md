# Issue status dropdown only shows default states, not custom states from config.yaml

When viewing an issue detail page, the status dropdown only shows the 3 hardcoded default states (Open, In Progress, Closed) instead of reading the custom statuses defined in the project’s config.yaml.

\*\*Expected:\*\* The status options should be populated from the \`statuses\` field in config.yaml.

\*\*Actual:\*\* Only the 3 default states are shown regardless of what is defined in config.yaml.
