# Project configuration page with advanced UI

## Overview

Transform the Settings page from read-only display to an editable configuration form with advanced UI components.

## Sections

### 1. Issue States

- Drag-to-reorder list of states
- Color picker for each state
- Default state selector dropdown
- Add/remove states with validation
- Live preview of state badges

### 2. Priority Levels

- Number of levels selector (1-10)
- Color picker for each priority level
- Live preview of priority badges

### 3. Custom Fields

- Card-based UI with collapsible sections
- Add/edit/remove custom fields
- Field properties:
  - Name (text input)
  - Type (dropdown: string, number, boolean, enum)
  - Required toggle
  - Default value
  - Enum values list (for enum type)

### 4. Defaults

- Key-value table editor
- Add/remove rows
- Autocomplete keys from custom field names

### 5. LLM Settings

- Checkbox: Auto-close issues when marked complete
- Checkbox: Update status to in-progress when work starts
- Checkbox: Allow direct edits to issue files

### 6. Project Version

- Semver input with validation
- Show current daemon version for reference

## UI Features

- Dirty state detection (unsaved changes indicator)
- Before unload warning for unsaved changes
- Save/Cancel buttons (sticky footer)
- Reset to defaults button
- Inline validation with error messages
- Loading/saving states

## New Components to Create

- `StateListEditor` - States management with drag-to-reorder
- `PriorityEditor` - Priority levels and colors
- `CustomFieldsEditor` - Custom fields list management
- `CustomFieldForm` - Single field editor (modal/inline)
- `DefaultsEditor` - Key-value table
- `LlmSettingsEditor` - LLM checkbox group
- `ColorPicker` - Hex color input with swatch preview

## Files to Modify

- `proto/centy.proto` - Sync from daemon
- `src/gen/centy_pb.ts` - Regenerate types
- `src/pages/Settings.tsx` - Major rewrite
- `src/pages/Settings.css` - New styles

1. Prevent deleting default_state (must change default first)
1. Semver format validation for version field
1. Required field indicators for custom field form
