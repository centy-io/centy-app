# Select project button does not reflect the current project from URL

## Problem

The "select project" button in the UI does not update its displayed value to match the project indicated by the current URL. When navigating directly to a URL for a specific project (e.g. via a link or browser reload), the button still shows the previously selected project or a default value instead of the one in the URL.

## Expected Behavior

The select project button should read the project identifier from the current URL and display the corresponding project name/value, keeping the UI in sync with the URL state at all times (including on initial load and navigation).

## Actual Behavior

The button displays a stale or default project selection regardless of the project in the URL. This causes a mismatch between the URL and the UI state.

## Acceptance Criteria

- [ ] On page load, the select project button reflects the project from the URL
- [ ] When navigating to a URL with a different project, the button updates accordingly
- [ ] The button remains in sync with the URL on browser back/forward navigation
