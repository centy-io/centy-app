# E2E: CI performance & parallelization improvements

Improve E2E test speed and efficiency in CI through parallelization, sharding, and smarter execution strategies.

## Tasks

- **Increase CI workers** – Currently CI runs with `workers: 1` (serial). Increase to 2–4 workers for parallel test execution.
- **Test sharding** – Split e2e tests across multiple CI jobs using Playwright's native `--shard` flag.
- **Shared auth state** – Use Playwright's storage state feature to save and reuse authenticated sessions, avoiding re-login per test.
- **Parallel test isolation** – Review shared state, database fixtures, and test data to prevent race conditions and flaky tests when running in parallel.
- **Reduce timeouts** – Analyze current test stability and reduce timeout values (currently 30s global, 10s expect) to surface slow tests faster.
- **Selective test runs on PR** – Implement test impact analysis or file dependency mapping to run only affected tests on PRs.
