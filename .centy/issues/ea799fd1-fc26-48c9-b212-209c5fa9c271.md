# Add debounce to search input

## Problem

The search input triggers a page reload/data refetch on every keystroke, causing excessive network requests and a poor user experience.

## Proposed Solution

Add debounce to the search input so that the refetch only fires after the user stops typing (e.g. 300–500ms delay).

## Acceptance Criteria

- [ ] Search input is debounced (300–500ms)
- [ ] Page does not reload/refetch on every keystroke
- [ ] Search still works correctly after debounce is applied
