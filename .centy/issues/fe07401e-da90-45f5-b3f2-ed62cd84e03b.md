# Open in X: redirect to worktree.io instead of handling internally

## Problem

"Open in X" feature currently handles the action internally. Should instead open a new browser tab and redirect to the corresponding worktree.io URL.

## Expected Behavior

Clicking "Open in X" opens a new tab pointing to the worktree.io location for that item. No internal handling required.

## Implementation Notes

- Remove internal "Open in X" handler logic
- Replace with `window.open(worktreeUrl, '_blank')` (or equivalent)
- Construct the correct worktree.io URL for the target item
