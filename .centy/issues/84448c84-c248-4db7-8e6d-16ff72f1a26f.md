# feat: show project item types in nav-group with capitalized first letter

## Problem

When fetching project item types from the API, the response includes types like `persona`, `Doc`, `Issue`, `story`. These should appear in the `nav-group-project` nav group with the first letter of each name capitalized.

## API Response Example

```json
{
  "success": true,
  "item_types": [
    { "name": "persona", "plural": "personas" },
    { "name": "Doc", "plural": "docs" },
    { "name": "Issue", "plural": "issues" },
    { "name": "story", "plural": "stories" }
  ],
  "total_count": 4
}
```

## Expected Behavior

The nav group should display:

- Personas
- Docs
- Issues
- Stories

(first letter capitalized, derived from the `plural` field of each item type)

## Acceptance Criteria

- [ ] Fetch item types from the project and display them in the `nav-group-project` nav group
- [ ] Each item type label should have its first letter capitalized (e.g. `personas` → `Personas`)
- [ ] All 4 item types (personas, docs, issues, stories) are shown
- [ ] Items appear in the order returned by the API
