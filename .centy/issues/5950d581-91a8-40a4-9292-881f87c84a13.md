# Add CSS for generic item create and edit form pages

The \`GenericItemCreate\` component (rendered at \`/{org}/{project}/{itemType}/new\`) and the \`GenericItemEditForm\` component (used inside \`GenericItemDetail\` when editing) use CSS classes that have no styles defined anywhere:

- \`.create-generic-item\` — outer page container
- \`.create-generic-title\` — page heading (h2)
- \`.create-generic-form\` — the form card
- \`.generic-item-edit-form\` — the inline edit form on the detail page

The create page also uses \`<Link className="secondary">\` inside \`.actions\` for the cancel button. The global CSS (from \`CreateIssue.css\`) only targets \`.actions button.secondary\`, not \`a.secondary\`, so the cancel link is unstyled.

## Goal

Add a \`GenericItemCreate.css\` file (imported in \`globals.css\`) that styles the create page, modeled after \`CreateDoc.css\`. Also add \`.generic-item-edit-form\` styles to \`GenericItemDetail.css\` so the edit form inside the detail page looks polished.

## Acceptance Criteria

- [ ] \`.create-generic-item\` — max-width 800px, centered, padded (matches \`.create-doc\`)
- [ ] \`.create-generic-title\` — large heading consistent with other create pages
- [ ] \`.create-generic-form\` — card-style background with border, border-radius, padding, shadow
- [ ] \`.create-generic-item .actions\` — flex row with gap, top border separator
- [ ] \`.create-generic-item .actions a.secondary\` — cancel link styled as secondary button
- [ ] \`.create-generic-item .actions button.primary\` — submit button styled as primary
- [ ] \`.generic-item-edit-form\` — styled edit form with consistent spacing and field grouping
- [ ] Responsive: reduce padding at ≤768px
- [ ] \`globals.css\` imports the new \`GenericItemCreate.css\`
