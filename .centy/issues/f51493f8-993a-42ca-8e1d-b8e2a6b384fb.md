# SSG: Add force-static to all pages missing static generation config

## Problem

Several pages are missing the `export const dynamic = 'force-static'` directive, either alongside existing `generateStaticParams` or on their own. Without this directive, Next.js may fall back to dynamic rendering unnecessarily.

## Pages to fix

### Pages with generateStaticParams but no force-static:

- app/organizations/[orgSlug]/page.tsx
- app/organizations/[orgSlug]/issues/page.tsx
- app/organizations/[orgSlug]/issues/new/page.tsx
- app/organizations/[orgSlug]/issues/[issueId]/page.tsx
- app/[organization]/[project]/config/page.tsx
- app/[organization]/[project]/docs/page.tsx
- app/[organization]/[project]/docs/[slug]/page.tsx
- app/[organization]/[project]/users/page.tsx
- app/[organization]/[project]/users/[userId]/page.tsx
- app/[organization]/[project]/assets/page.tsx
- app/[organization]/[project]/page.tsx
- app/[organization]/[project]/issues/page.tsx
- app/[organization]/[project]/issues/new/page.tsx (also tracked in #110)
- app/[organization]/[project]/issues/[issueId]/page.tsx
- app/[...path]/page.tsx

### Static pages missing force-static:

- app/organizations/new/page.tsx (also tracked in #105)
- app/organizations/page.tsx
- app/project/init/page.tsx (also tracked in #106)
- app/project/config/page.tsx (remove use client from page level, add force-static)
- app/assets/page.tsx (remove use client from page level, add force-static)

## Solution

Add export const dynamic = 'force-static' to all listed pages. For pages with use client at page level, remove it. Closes #105, #106, and #110.
