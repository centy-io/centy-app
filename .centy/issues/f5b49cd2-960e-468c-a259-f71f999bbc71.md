# Add soft delete and hard delete options when deleting an item

When a user deletes an item, the current flow only has a single delete action. We want to present two distinct options:

- **Soft delete**: Moves the item to a "deleted" / archived state, keeping it recoverable. (Maps to the existing `SoftDeleteItem` RPC.)
- **Hard delete**: Permanently removes the item and all associated data with no ability to recover. (Maps to the existing `DeleteItem` RPC.)

The delete confirmation UI (e.g. a dialog or action menu) should surface both buttons clearly so the user can make an intentional choice.
