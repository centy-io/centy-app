# Cascade untrack projects when untracking organization

When attempting to untrack an organization that has projects, instead of blocking with `ORG_HAS_PROJECTS` error, offer the user a cascade option that removes tracking for all the organization's projects as well.

## Current behavior

The dialog shows the error: "Organization has N projects. Reassign or remove them first." with no way to proceed.

## Desired behavior

When the `ORG_HAS_PROJECTS` error is returned, show a confirmation prompt:

- Explain that untracking the org will also remove tracking for its N projects
- On confirm, retry the `deleteOrganization` call with `cascade: true`

## Technical context

The daemon already supports cascade deletion via the `cascade` flag on `DeleteOrganizationRequest`. This is purely a frontend change.

## Files to reference

- `centy-app/components/organizations/OrganizationsList/useOrganizationsList.ts` (untrack org handler, error handling)
- Daemon proto: `DeleteOrganizationRequest` has a `cascade` boolean field
