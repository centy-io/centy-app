# E2E: Test architecture & quality improvements

A collection of improvements to the E2E test architecture, patterns, and tooling to make tests more maintainable, reliable, and efficient.

## Architecture & Patterns

- **Page Object Model (POM)** – Refactor e2e tests to use POM pattern for better maintainability, reusability, and cleaner test code. Encapsulate page interactions in dedicated classes.
- **API mocking layer** – Implement an API mocking layer to mock backend responses for UI-focused tests. Enables faster, more isolated testing without backend dependencies.
- **Test data factories** – Create test data factories to generate consistent fixtures across tests.
- **Database seeding** – Implement database seeding to preload known state before tests run, reducing setup time.
- **Component testing** – Add Playwright component testing for UI unit tests to catch bugs earlier.

## Infrastructure

- **Containerized test environment** – Set up Docker-based environment for consistent cross-platform test execution (identical results between local and CI).
- **Cache browser binaries in CI** – Cache Playwright browser installations between CI runs to reduce startup time.
- **Parallelized visual tests with Docker matrix** – Run visual tests in parallel across multiple Docker containers using matrix strategy in CI.

## DX & Tooling

- **Test retries with flake detection** – Implement intelligent retries to distinguish genuine failures from flaky tests. Track and report flake rates.
- **Smoke test script** – Add `pnpm e2e:fast` that runs only critical-path tests for quick feedback during development.
- **Test tagging** – Implement `@smoke`, `@slow`, `@critical` tags to enable selective test execution based on context.
- **Playwright UI mode** – Document and add npm script for Playwright UI mode for interactive test debugging.
- **Coverage reports** – Implement code coverage reporting for e2e tests to identify gaps in test coverage.
