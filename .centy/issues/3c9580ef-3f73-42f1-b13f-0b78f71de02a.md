# Show daemon update available indicator in header-controls

## Problem

When a newer version of the centy daemon is available, users have no in-app indication. They must discover updates manually, which leads to users running outdated daemon versions without knowing.

## Goal

Add a “New version available” indicator to the \`header-controls\` area in \`Header.tsx\`. When the daemon is connected and a newer version exists, show a badge/button next to the \`DaemonStatusIndicator\`. Clicking it opens a dialog or tooltip that displays the bash install command for the new version.

## How it works

1. The daemon exposes \`GetDaemonInfo\` (returns \`DaemonInfo.version\` — current running version, e.g. \`“0.3.1”\`)
1. The app fetches the latest available daemon version (e.g. from a release endpoint or a known source)
1. If latest > current, show an update badge in \`header-controls\`
1. Clicking the badge shows the bash install command so the user can copy and run it to upgrade

## Affected areas

- \`components/providers/DaemonStatusProvider.hooks.ts\` — fetch latest version alongside \`getDaemonInfo\`, expose \`daemonVersion\` and \`latestDaemonVersion\` via context
- \`components/providers/DaemonStatusProvider.tsx\` — add \`daemonVersion\` and \`latestDaemonVersion\` to context value and type
- \`components/providers/DaemonStatusProvider.types.ts\` — extend \`DaemonStatusContextType\`
- \`components/shared/DaemonUpdateBadge.tsx\` — new component rendered inside \`header-controls\` when update is available; on click shows install command in a dialog/popover with a copy button
- \`components/layout/Header/Header.tsx\` — render \`<DaemonUpdateBadge />\` inside \`.header-controls\`

## Acceptance Criteria

- [ ] When daemon is connected and a newer version is available, a visible “Update available” badge appears in the header controls
- [ ] Clicking the badge opens a dialog/popover showing the bash install command for the new version
- [ ] The install command is displayed in a code block with a copy-to-clipboard button
- [ ] When the daemon is on the latest version (or disconnected/demo), the badge is not shown
- [ ] Unit tests cover the version comparison logic and badge visibility
- [ ] Unit tests cover the \`DaemonUpdateBadge\` component rendering and copy behavior
