# Show git repository warning in UI for project initialization

When initializing a new project via the app UI:

1. Check if the selected directory is inside a git repository
1. If not inside a git repo, show a warning dialog to the user
1. Allow user to proceed with a clear acknowledgment that this is not recommended
1. Display a hint/tooltip explaining why initializing inside a git repo is recommended

This ensures users are aware when creating .centy folders outside of git repositories.

## Acceptance Criteria

- [ ] Add git repository detection in project initialization flow
- [ ] Show warning dialog when not in a git repo
- [ ] Add option to proceed anyway with acknowledgment
- [ ] Include helpful tooltip/hint explaining the recommendation
