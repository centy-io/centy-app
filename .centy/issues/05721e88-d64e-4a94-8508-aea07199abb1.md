# Add soft delete and hard delete options when deleting an item

When a user triggers a delete action on an item, present two options instead of a single confirmation:

- **Soft delete** — marks the item as deleted (e.g. sets a `deletedAt` timestamp) but retains it in the database. The item is hidden from normal views but can be recovered.
- **Hard delete** — permanently removes the item from the database with no possibility of recovery.

## Acceptance Criteria

- The delete confirmation dialog/modal shows two distinct actions: “Archive” (soft delete) and “Delete permanently” (hard delete).
- Soft-deleted items are excluded from all standard list/query results.
- A way to view and restore soft-deleted items exists (e.g. a trash/archive view or a restore action).
- Hard delete requires an additional confirmation step or warning indicating the action is irreversible.
- Both paths are covered by unit and/or integration tests.
