# Fail build when required environment variables are missing

## Problem

Currently, if a required environment variable is missing at build time, the build succeeds silently. This can lead to runtime errors or misconfigured deployments that are hard to debug.

## Goal

The build should fail fast and loudly when any required environment variable is not defined, surfacing the issue at build time rather than at runtime.

## Proposed Solution

Add a build-time environment variable validation step that:

1. Defines a list of all required environment variables
2. Checks each one at the start of the build
3. Throws an error with a clear message listing all missing variables if any are absent

This can be implemented in `next.config.ts` or as a pre-build script.

## Acceptance Criteria

- [ ] A canonical list of required environment variables is defined in the codebase
- [ ] The build fails with a descriptive error when any required env var is missing
- [ ] The error message clearly lists which variables are missing
- [ ] The validation runs before any compilation/bundling occurs
- [ ] CI pipelines surface the failure clearly
