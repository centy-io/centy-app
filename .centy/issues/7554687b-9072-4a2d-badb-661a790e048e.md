# feat: support comments from daemon

The daemon added \`comments\` as a built-in item type in centy-io/centy-daemon#235. The CLI should expose this so users can manage comments from the terminal.

## Background

Comments use existing generic item RPCs (\`CreateItem\`, \`ListItems\`, \`UpdateItem\`, \`DeleteItem\`) with \`item_type: "comments"\`. They have UUID identifiers and custom fields: \`item_id\` (parent item UUID), \`item_type\` (parent item type), and \`author\`. All optional features are disabled (no display number, priority, soft delete, etc.).

To fetch all comments for a parent item, use \`ListItems\` with \`filter: {"customFields": {"item_id": "<uuid>"}}\`.

## Tasks

- [ ] \`centy comment add <item-id> --author <author> <body>\` — create a comment on an item
- [ ] \`centy comment list <item-id>\` — list all comments for an item
- [ ] \`centy comment update <comment-id> <body>\` — update a comment
- [ ] \`centy comment delete <comment-id>\` — delete a comment
