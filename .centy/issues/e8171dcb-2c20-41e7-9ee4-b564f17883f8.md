# Merge create doc and create issue hooks into a single create item hook

The daemon currently has separate hooks for creating docs and creating issues. These should be unified into a single `create item` hook that handles both item types.

**Motivation:** Both hooks share the same fundamental pattern — creating a new item in the workspace. Having two separate hooks creates duplication in hook configuration and makes it harder for users to apply common logic (e.g. templating, validation, notifications) across all item creation events.

**Proposed change:** Replace the `create doc` and `create issue` hooks with a single `create item` hook. The hook payload should include an item type field (e.g. `kind: "doc" | "issue"`) so consumers can still distinguish between the two if needed.
