# Fix visual test consistency across environments

Visual/screenshot tests currently fail across environments due to font rendering differences between macOS and Linux (CI). This causes false positives in the pre-push hook, forcing developers to use `--no-verify`.

## Root Cause

Font rendering differs between Linux (CI) and macOS (local). Screenshots generated on Linux don't match when compared on macOS — sub-pixel rendering and font hinting vary by OS.

## Solution

### 1. Bundle self-hosted web fonts

Use `@fontsource` packages (e.g. Inter, Noto Sans) instead of system fonts. Include exact WOFF2 font files in the project — no external CDN or system fonts.

```css
@font-face {
  font-family: 'App Font';
  src: url('/fonts/inter.woff2') format('woff2');
  font-display: block;
}

body {
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
  text-rendering: optimizeLegibility;
}
```

### 2. Docker-based visual testing

Run visual tests in a Linux Docker container locally to match CI environment.

```json
{
  "e2e:visual": "playwright test --grep @visual --project=chromium",
  "e2e:visual:docker": "docker compose -f e2e/docker-compose.yml run --rm playwright pnpm e2e:visual",
  "e2e:visual:update": "docker compose -f e2e/docker-compose.yml run --rm playwright pnpm e2e:visual -- --update-snapshots"
}
```

### 3. Update Playwright config

Support `RUN_VISUAL_TESTS=true` env var to enable visual tests in CI when explicitly triggered. Add a separate `visual-tests.yml` GitHub workflow (manual dispatch + triggered on screenshot file changes).

### 4. Regenerate snapshots

Remove old macOS-generated snapshots and regenerate using Docker (matches CI environment).

## Acceptance Criteria

- [ ] All fonts bundled locally (no CDN)
- [ ] Font rendering identical on macOS, Linux, and Windows
- [ ] Visual tests pass both locally (via Docker) and in CI
- [ ] Pre-push hook works without `--no-verify`
- [ ] `pnpm e2e:visual:docker` and `pnpm e2e:visual:update` scripts work
- [ ] Separate CI workflow for visual test validation
