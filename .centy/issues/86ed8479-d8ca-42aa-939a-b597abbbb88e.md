# Remove genericItemToIssue adapter and migrate consumers to GenericItem

The `lib/genericItemToIssue.ts` adapter is a compatibility shim left over from the proto migration that replaced the `Issue` type with the generic `GenericItem` model. It converts `GenericItem` → legacy `Issue` so existing UI components continue to work without changes.

It has 10 call sites across the codebase:

- `components/issues/IssuesList/hooks/useIssuesData.ts`
- `components/issues/AggregateIssuesList/hooks/useAggregateIssues.ts`
- `components/issues/IssueDetail/hooks/useStatusChange.ts`
- `components/issues/IssueDetail/hooks/useIssueDetail.ts`
- `components/issues/IssueDetail/hooks/performSave.ts`
- `components/shared/AddLinkModal/fetchIssueEntities.ts`
- `components/organizations/OrgIssueDetail/hooks/useOrgIssueFetch.ts`
- `components/organizations/OrgIssuesList/hooks/useOrgIssues.ts`
- `components/organizations/OrgIssueDetail/hooks/performSaveIssue.ts`

## Goal

Migrate all consumers to work directly with `GenericItem` (or a subset of its fields), then delete `lib/genericItemToIssue.ts` and `lib/genericItemToIssue.spec.ts`. The legacy `Issue` interface in `gen/centy_pb.ts` can be removed once nothing references it.

## Context

`genericItemToDoc` was the same pattern and was already removed — that was straightforward because it had a single consumer (`fetchDocEntities.ts` → `filterAndMapDocs.ts`). This one requires updating all issue-related components.
