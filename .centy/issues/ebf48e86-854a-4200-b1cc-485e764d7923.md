# Simplify AddLinkModal entity search by removing artificial type filter

The `useEntitySearch` hook in `components/shared/AddLinkModal/` splits item search into two separate fetches — one for issues (`fetchIssueEntities`) and one for docs (`fetchDocEntities`) — controlled by a `targetTypeFilter` ('issue' | 'doc') toggle.

This is an artificial constraint. Since links can point to any item, there's no reason to require the user to select a type before searching. The current design:

- Forces the user to pick a type filter before searching, rather than seeing all linkable items at once
- Makes two round trips to see everything
- Adds unnecessary state (`targetTypeFilter`), toggle UI, and two separate fetch functions

**Proposed fix:** Replace the two fetch functions with a single `listItems` call without a hardcoded `itemType` filter (or passing all types), a unified `filterAndMap` function, and remove the `targetTypeFilter` state and UI toggle entirely.

If the item list is too large for a single fetch, the right solution is server-side search, not a client-side type toggle.
