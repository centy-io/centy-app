# Enable @typescript-eslint/no-confusing-void-expression and fix violations

Enable the `@typescript-eslint/no-confusing-void-expression` rule (currently set to `'off'` in `eslint.config.js`) and fix all violations.

The rule flags arrow function shorthands that return void expressions, e.g. `onClick={() => handler()}` which looks like it returns a value but doesn't.

**Options for fixing violations:**

1. Add braces: `onClick={() => { handler() }}` — ESLint's default autofix, but prettier expands these to 3 lines each, which pushes several functions over the `max-lines-per-function` limit (70 lines).
2. Use `void` operator with `ignoreVoidOperator: true`: `onClick={() => void handler()}` — keeps it single-line, explicit about discarding the return value, but requires configuring the rule.
