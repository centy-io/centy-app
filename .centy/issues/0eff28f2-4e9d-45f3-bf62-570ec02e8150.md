# Consolidate color definitions: remove colors.ts, use only CSS custom properties

## Problem

Color values are currently duplicated across two files:

- `styles/colors.ts` — TypeScript object of rgba() values, imported by TS/TSX files that need colors programmatically (inline styles, dynamic styling)
- `styles/index.css` — CSS custom properties that mirror the same rgba values (and hold all other design tokens)

The comment in `colors.ts` even acknowledges this split: _"CSS files reference these via CSS custom properties defined in styles/index.css."_ This means any color change must be made in two places, and the two can drift out of sync.

## Goal

Eliminate `styles/colors.ts` entirely. All colors live only as CSS custom properties in `styles/index.css`.

## Approach

For components that currently import `colors` for inline styles or dynamic JS usage, switch to reading CSS variables at runtime via `getComputedStyle(document.documentElement).getPropertyValue('--color-...')`, or pass CSS variable strings directly in inline styles where React supports it (e.g. `style={{ color: 'var(--color-danger)' }}`).

Note: the solid hex colors in `colors.ts` (`stateOpen`, `stateClosed`, priority colors, `dangerRed`, `fallback`) that are not yet in `index.css` will need to be added as CSS custom properties first.
