# Improve Generic Item Detail Page Styling

Currently the generic item detail page (`GenericItemDetail`, `GenericItemView`, `GenericItemDetailHeader`) renders with CSS class names like `.generic-item-detail`, `.generic-item-view`, `.generic-item-view-title`, `.generic-item-id-display`, `.generic-item-field-display`, `.generic-item-body`, etc. — but **no CSS rules exist** for these classes. The page borrows `.doc-detail` and `.doc-header` classes from `DocDetail.css` for some elements, but the view-specific classes have no styles, causing the detail page to render as unstyled flat text.

## Goal

Add CSS for the generic item detail page, modeled after `DocDetail.css`, so the detail view looks polished and consistent with the rest of the app.

## Acceptance Criteria

- [ ] `.generic-item-detail` — max-width 800px, centered, padded (same as `.doc-detail`)
- [ ] `.generic-item-view` — readable layout with consistent spacing between sections
- [ ] `.generic-item-view-title` — large heading, consistent with doc detail title style
- [ ] `.generic-item-id-display` — mono font, muted color, small size (matches ID style on cards)
- [ ] `.generic-item-status-display` — inline badge or labeled row with status pill
- [ ] `.generic-item-field-display` — labeled row for each custom field, with label/value styling
- [ ] `.generic-item-body` — proper spacing above the markdown body content
- [ ] `.generic-item-meta` — muted small timestamp row at the bottom
- [ ] Header actions (Edit / Save / Cancel / Delete buttons) use `.doc-header` pattern already in place — verify they render correctly with existing `DocDetail.css` rules
- [ ] `deleted-item-banner` and `restore-btn` are already styled via `GenericItems.css` — verify they appear correctly on the detail page too
- [ ] Responsive: reduce padding and font sizes at ≤768px
