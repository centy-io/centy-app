# Add PurgeCSS integration (like Knip for CSS)

Add a PurgeCSS-like integration that detects and removes unused CSS rules from the codebase.

Similar to how Knip identifies unused JS/TS exports, files, and dependencies, this tool would:

- Scan component and page files to determine which CSS class names are actually used
- Flag or remove unused CSS rules from stylesheets
- Integrate into the dev/build workflow (e.g. as a CLI command or build step)
- Optionally report unused styles without removing them (dry-run mode)

This would help reduce stylesheet bloat as the codebase grows.
